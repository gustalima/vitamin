GENERAL USAGE NOTES

  • This software combines several tools for protein modeling, such as BLAST search for templates, Modeller, procheck and PyMOL. All the process, from searching templates to model evaluation and refining can be done in the ViTaMIn.

  • You WILL need a valid Modeller license (free for academics, https://salilab.org/modeller/registration.html). ViTaMIn installer will not install Modeller, but you can find Installation Notes for it on official website .

Installing ViTaMIn under Linux

Dependecies

ViTaMIn depends on the following packages:

  • Modeller (9.9 or newer)
  • pymol (0.99 or newer, 1.3+ recommended)
  • procheck
  • matplotlib (1.2 or newer)
  • biopython (1.6 or newer)
  • GTK3+
  • python (2.7.x)

System Requirements

The major job is done by Modeller, and as is, it needs a i686 or amd64 processor to work. The faster your processor, the faster your job is done. There is no minimum requirements. Internet connection is required for BLAST search engine only, but you can use it without one if you have sequences and pdbs on your disk .

Ubuntu 12.04 or newer

Installing ViTaMIn in Ubuntu is a straight process using the provided script (install). The script checks and install dependencies automatically (may require root permission) and set procheck on your machine. Note that the procheck version is the most update to date provided by Roman Laskowski, with a simple change in the gets function (to fgets) in order to use newer versions of the cc compiler. You can choose not to install procheck, if you already have a working version with the 'procheck' alias.

For Installion, use the command in download folder: Easy way: open terminal (install git if you don't have it already) git clone http://github.com/gustalima/vitamin-stable cd vitamin-stable ./install

Without git: Download Vitamin.tar.gz and procheck.tar.gz and install run the install file with the following command ./install

Other distros

For all other distros, you will need to get dependencies by yourself. It's not that hard, and most distros have them in their repositories. Once your system dependencies are OK, untar the tarball file provided in any folder and run from terminal. Any missing dependencies you may have will be alerted in your terminal .
